# Security policy

## Supported documentation

Security-sensitive deployment guidance is maintained on the `main` branch. Older tagged versions may contain obsolete package or platform details.

## Reporting a security problem in this guide

Do not open a public issue for a vulnerability that could put a deployed user at immediate risk. Use the repository owner’s private security advisory/reporting channel and include:

- affected chapter/template,
- the unsafe behavior,
- preconditions,
- impact,
- a safer correction if known.

This document is educational material, not a guarantee that any deployment is secure. Review every configuration for your threat model and environment.
